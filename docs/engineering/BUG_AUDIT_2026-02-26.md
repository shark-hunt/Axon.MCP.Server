# Axon.MCP.Server Bug Audit (2026-02-26)

Scope: repository-wide static review (Python backend), compile checks, targeted reliability/security inspection.

## Audit Log

- Ran syntax compile pass across `src/` and `tests/` using `py_compile`.
- Ran targeted source scans for exception swallowing and async misuse patterns.
- Performed manual deep inspection of parser, locking, API, and security utility modules.

## Findings

- [x] **B001 — Syntax error blocks module compilation**  
  **Severity:** Critical  
  **File:** `src/parsers/csharp_minimal_api.py:2`  
  **Evidence:** File starts with an indented function at top-level, causing `IndentationError` (`unexpected indent`).  
  **Impact:** Breaks compile checks and can break packaging/static tooling.  
  **Resolution (2026-02-26):** Removed orphaned malformed file `src/parsers/csharp_minimal_api.py` (duplicate logic already exists in `csharp_parser.py`). Verified `py_compile` passes for all `src/` and `tests/` modules.

- [x] **B002 — Event loop handling bug in sync parser wrapper**  
  **Severity:** High  
  **File:** `src/parsers/hybrid_parser.py:57-72`  
  **Evidence:** `parse()` raises `RuntimeError` when already inside a running loop, then immediately catches all `RuntimeError` and continues to `asyncio.run(...)`.  
  **Impact:** In async contexts this can still call `asyncio.run()` and fail with loop-related runtime errors.  
  **Resolution (2026-02-26):** Refactored guard logic to only call `asyncio.run()` when no running loop exists; now explicitly fails fast when called from an active loop.

- [x] **B003 — Invalid async context manager usage in DB health check**  
  **Severity:** High  
  **File:** `src/database/connection_pool.py:93`  
  **Evidence:** `async with self.get_session() as session:` uses coroutine-returning method without awaiting.  
  **Impact:** Health check can fail at runtime with context manager/type errors.  
  **Resolution (2026-02-26):** Updated health check to await session acquisition before entering async context manager.

- [x] **B004 — Redis lock can release another worker’s lock**  
  **Severity:** High  
  **File:** `src/workers/distributed_lock.py`  
  **Evidence:** Lock value was constant (`"locked"`) and release was unconditional `delete(lock_key)` with no ownership token verification.  
  **Impact:** If lock expires/reacquires between operations, one worker may delete another worker’s lock.  
  **Resolution (2026-02-26):** Reworked lock acquisition to store per-lock UUID owner tokens (`SET NX EX token`) and changed release to Lua compare-and-delete (`get == token -> del`), preventing cross-worker unlock races; refreshed unit coverage in `tests/unit/test_distributed_lock.py`.

- [x] **B005 — Advisory lock timeout parameter is unused + raw SQL interpolation**  
  **Severity:** Medium  
  **File:** `src/utils/repository_lock.py`  
  **Evidence:** `timeout_seconds` was accepted but not enforced; SQL was built via f-string instead of bound params.  
  **Impact:** Misleading API contract and avoidable SQL construction risk.  
  **Resolution (2026-02-26):** Added bounded retry loop to honor `timeout_seconds` when acquiring advisory locks and replaced interpolated SQL with parameterized `text(..., {"lock_id": ...})` calls for lock/unlock/check operations; added unit coverage in `tests/unit/test_repository_lock.py`.

- [x] **B006 — Secret masking can expose too much for short strings**  
  **Severity:** Medium  
  **File:** `src/utils/security.py`  
  **Evidence:** Formula `'*' * (len(data) - visible_chars * 2)` could become negative for short values; output concatenated overlapping prefix/suffix.  
  **Impact:** Sensitive tokens shorter than `2*visible_chars` could be under-masked.  
  **Resolution (2026-02-26):** Updated `mask_sensitive_data()` to fully mask short values, fully mask for non-positive `visible_chars`, and only reveal bounded prefix/suffix when enough length exists; added explicit unit coverage in `tests/unit/test_security_masking.py`.

- [x] **B007 — CORS defaults conflict with credentialed requests**  
  **Severity:** Medium  
  **Files:** `src/config/settings.py`, `src/api/main.py`, `.env.example`, `docs/engineering/SECURITY_DEFAULTS_MIGRATION_2026-02-26.md`  
  **Evidence:** Default `api_cors_origins=["*"]` while `allow_credentials=True`.  
  **Impact:** Browsers reject wildcard origin for credentialed requests; cookie auth can break and config is insecure by default.  
  **Resolution (2026-02-26):** Switched default CORS origins to explicit localhost entries, made credentials conditional (`allow_credentials=False` when wildcard is configured), and documented migration/override behavior.

- [x] **B008 — Request-flow statistics contain placeholder zeros**  
  **Severity:** Medium  
  **File:** `src/mcp_server/tools/architecture.py`  
  **Evidence:** `cross_service_calls` and `event_publications` hardcoded to `0` with TODO comments.  
  **Impact:** Misleading architecture trace output and weak observability.  
  **Resolution (2026-02-26):** Replaced placeholders with DB-backed aggregations scoped to traversed flow symbols (`ApiEndpointLink`/`OutgoingApiCall` for cross-service calls and `PublishedEvent`/`EventLink` for linked publications), and surfaced both metrics in output.

- [x] **B009 — Insecure auth default for MCP transport**  
  **Severity:** High  
  **Files:** `src/config/settings.py`, `src/api/main.py`, `.env.example`, `docs/engineering/SECURITY_DEFAULTS_MIGRATION_2026-02-26.md`  
  **Evidence:** `mcp_auth_enabled: bool = False` by default.  
  **Impact:** MCP endpoints can run without auth unless explicitly hardened.  
  **Resolution (2026-02-26):** Changed default to `true`, updated environment template, and added startup warning for `mcp_transport=http` with `mcp_auth_enabled=false` to provide explicit secure onboarding guidance.

- [x] **B010 — Insecure TLS verification default for Azure DevOps**  
  **Severity:** Medium  
  **Files:** `src/config/settings.py`, `.env.example`, `docs/engineering/SECURITY_DEFAULTS_MIGRATION_2026-02-26.md`  
  **Evidence:** `azuredevops_ssl_verify: bool = False` default.  
  **Impact:** Encourages insecure TLS behavior and MITM exposure unless overridden.  
  **Resolution (2026-02-26):** Changed default to `true`, updated environment template, and documented explicit opt-out instructions for trusted self-signed environments.

### Rolling review follow-ups

- [x] **R001 — Roslyn process health was reporting stale placeholder response timing**  
  **Severity:** Low  
  **File:** `src/parsers/roslyn/process_manager.py`  
  **Evidence:** `ProcessHealth.last_response_time` always returned `0.0`, despite active request handling.  
  **Impact:** Health/observability views cannot distinguish responsive vs degraded Roslyn round-trip latency.  
  **Resolution (2026-02-26):** Added `time.perf_counter()` round-trip timing in `send_request()`, persisted `_last_response_time`, surfaced in both running/non-running `get_health()` responses, and added regression test `tests/unit/test_roslyn_process_manager.py::test_send_request_tracks_last_response_time`.

- [x] **R002 — Test bootstrap failed early because settings required prod secrets in test env**  
  **Severity:** Medium  
  **Files:** `src/config/settings.py`, `tests/unit/test_security_defaults.py`  
  **Evidence:** `pytest` collection crashed in `tests/conftest.py` when `get_settings()` instantiated `Settings()` without `gitlab_token`, `api_secret_key`, and `jwt_secret_key` in test env.  
  **Impact:** Local/CI test runs can fail before executing tests unless unrelated production secrets are injected.  
  **Resolution (2026-02-26):** Added environment-aware model validation that still enforces critical secrets outside `testing`, while allowing explicit test runs to omit them; added regression tests for both behaviors.

- [x] **R003 — Roslyn process manager used deprecated naive UTC timestamps**  
  **Severity:** Low  
  **Files:** `src/parsers/roslyn/process_manager.py`, `tests/unit/test_roslyn_process_manager.py`  
  **Evidence:** Test runs emitted deprecation warnings for `datetime.utcnow()` usage in process start/request/uptime tracking paths.  
  **Impact:** Future Python versions may remove support; warning noise obscures real regressions.  
  **Resolution (2026-02-26):** Migrated to timezone-aware UTC timestamps using `datetime.now(UTC)` and updated tests to match.

- [x] **R004 — Heartbeat-task cancellation path could trigger unawaited coroutine warnings in test/runtime edges**  
  **Severity:** Low  
  **Files:** `src/parsers/roslyn/process_manager.py`  
  **Evidence:** `stop()` blindly called `self._heartbeat_task.cancel()`; when task-like mocks/adapters return awaitables from `cancel()`, warning noise (`coroutine was never awaited`) appears.  
  **Impact:** Warning noise can mask legitimate failures and weakens shutdown hygiene in non-standard task wrappers.  
  **Resolution (2026-02-26):** Hardened cancellation path to await `cancel()` results when awaitable (`inspect.isawaitable`) and replaced bare `except:` blocks around process kill with `except Exception`.

- [x] **R005 — FastAPI lifecycle hooks were using deprecated `@app.on_event` API**  
  **Severity:** Low  
  **Files:** `src/api/main.py`  
  **Evidence:** Test runs emitted deprecation warnings indicating `on_event` startup/shutdown handlers should migrate to lifespan handlers.  
  **Impact:** Warning noise obscures real regressions and risks breakage in future FastAPI versions.  
  **Resolution (2026-02-26):** Migrated startup/shutdown logic into a single `lifespan` async context manager and wired it into `FastAPI(..., lifespan=...)`; also switched request latency timing to `time.perf_counter()` for monotonic high-resolution duration metrics.
